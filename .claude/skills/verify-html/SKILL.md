# Skill: Verify HTML

Verify that generated HTML correctly represents the source markdown. Read both files and run structural checks.

## Inputs

- The markdown source and HTML output are in the same directory (the agent's update or item directory)
- The agent provides the paths to both files

## Checks

Read both files, then verify:

### 1. Headings preserved
- Every `#`, `##`, `###` heading in the markdown appears as `<h1>`, `<h2>`, `<h3>` in the HTML
- Heading text matches (after stripping markdown link syntax)

### 2. Links converted
- Every `[text](url)` link in the markdown exists as an `<a href="url">` tag in the HTML
- No literal `[text](url)` markdown link syntax remains in the HTML

### 3. Due-diligence flags styled
- Every `_(...)_` flag in the markdown has a corresponding `<em style="color:#e67700` tag in the HTML
- Count of flags in markdown matches count of orange-styled `<em>` tags in HTML

### 4. Callout blocks present
- If markdown has `**What happened:**`, the HTML has a div with `background-color:#edf2ff`
- If markdown has `**What to watch:**`, the HTML has a div with `background-color:#edf2ff`
- Continuation paragraphs (text after the what-block, before the next heading) are inside the callout div, not outside

### 5. No raw markdown in HTML
- No literal `**text**` (unprocessed bold) in the HTML
- No literal `*text*` (unprocessed italic) in the HTML outside of `<em>` tags
- No literal `_(...)_` (unprocessed due-diligence flags) in the HTML

### 6. Footer present
- HTML contains `Generated by Claude Stalk` in a footer div

### 7. Category-specific checks

**Audio:**
- Metadata pills are present (spans with `background-color:#d0ebff` for Source)
- Item cards have `background-color:#f8f9fa` wrapping
- Key moment list items have `border-left:2px solid #dee2e6`
- Timestamp links have `font-family:Courier New`

**X:**
- Metadata table has label cells with `width:200px`
- Skipped content section has box divs with `background-color:#f8f9fa`
- Body paragraphs have explicit `<p>` styling

**Webpage:**
- Article cards have `background-color:#f8f9fa` wrapping (same as audio cards)
- Source metadata pill present (span with `background-color:#d0ebff`)
- No `**Published**:` pill (webpages don't include publish date in metadata line)
- No timestamp deep links (`font-family:Courier New` should NOT appear)
- Body paragraphs inside cards use `<p>` styling
- Optional Guest(s) pill with `background-color:#fff3bf` (only for interviews)

## Report

```
## HTML Verification

- [PASS/FAIL] Headings: N/N matched
- [PASS/FAIL] Links: N/N converted
- [PASS/FAIL] Due-diligence flags: N/N styled
- [PASS/FAIL] Callout blocks: present/missing
- [PASS/FAIL] Raw markdown: none found / N instances
- [PASS/FAIL] Footer: present/missing
- [PASS/FAIL] Category checks: all passed / N issues

Verdict: PASS / FAIL (regenerate)
```

If any check fails, report the specific issue and recommend regeneration with `python scripts/md-to-html.py`.
